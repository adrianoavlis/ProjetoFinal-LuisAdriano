✅ Requisitos Funcionais (RF)

São as funcionalidades que o sistema deve oferecer:

Coleta de Dados

RF01: O sistema deve coletar dados de preços da cesta básica em fontes oficiais (ex.: DIEESE, Conab, Procon, IBGE).

RF02: O sistema deve permitir a importação de dados externos via CSV/Excel ou API (quando disponível).

RF03: O sistema deve suportar scraping controlado e ético de dados de supermercados (quando autorizado).

Armazenamento e Organização

RF04: O sistema deve armazenar os dados em um banco de dados estruturado (ex.: MySQL/PostgreSQL).

RF05: O sistema deve manter o histórico de preços por produto, mercado e período.

Processamento e Análise

RF06: O sistema deve calcular indicadores como:

preço médio, maior e menor preço por item;

evolução histórica por produto/cidade;

mercado mais vantajoso para a compra.

RF07: O sistema deve permitir comparações regionais (entre capitais ou bairros).

RF08: O sistema deve gerar relatórios analíticos (PDF/Excel) com gráficos.

Visualização e Interatividade

RF09: O sistema deve exibir dashboards com gráficos (linha, barra, dispersão).

RF10: O sistema deve oferecer filtros por período, produto, mercado e região.

RF11: O sistema deve disponibilizar uma tabela interativa para consulta.

Gestão e Usuários

RF12: O sistema deve permitir autenticação de usuários (login).

RF13: O sistema deve diferenciar permissões (ex.: administrador, pesquisador).

RF14: O sistema deve permitir exportar relatórios sob demanda.

⚙️ Requisitos Não Funcionais (RNF)

Definem qualidades e restrições do sistema:

Desempenho

RNF01: O sistema deve processar consultas em até 3 segundos para bases com até 1 milhão de registros.

RNF02: O scraping não deve sobrecarregar sites (respeitar rate limiting).

Segurança

RNF03: Os dados de login devem ser protegidos por hash seguro (ex.: bcrypt, SHA-256).

RNF04: O sistema deve atender à LGPD, evitando coleta de dados pessoais.

RNF05: Conexões devem ser feitas com HTTPS/TLS.

Usabilidade

RNF06: O sistema deve ter interface responsiva (desktop e mobile).

RNF07: O design deve seguir boas práticas de acessibilidade (ex.: contraste, texto alternativo).

Confiabilidade

RNF08: O sistema deve realizar backup automático diário dos dados.

RNF09: O sistema deve registrar logs de erros e acessos para auditoria.

Manutenibilidade

RNF10: O código deve ser documentado e versionado em GitHub/GitLab.

RNF11: O sistema deve possuir testes automatizados (unitários, integração, sistema).

Portabilidade

RNF12: O sistema deve ser executável em ambiente Docker para facilitar a implantação.

RNF13: Deve suportar bancos de dados SQL compatíveis (MySQL, PostgreSQL).
